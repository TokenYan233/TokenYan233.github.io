🚀 三种工作模式

提供了三种不同的工作模式，适应不同的使用场景。你可以根据自己的需求选择合适的模式。

📋 模式 0：仅查询链接（适合高级用户和开发者）

这个模式的用途：

这个模式只会查询 Chrome 的版本信息和下载链接，完全不会下载任何文件。适合那些想要：

  • 🔗 获取下载链接，用其他工具（比如迅雷、IDM）来下载的用户
  • 📊 仅查看最新版本信息的用户
  • 🔧 想把获取到的链接传递给其他程序或脚本处理的开发者
  • 📝 需要记录 Chrome 版本历史的用户

具体怎么使用：

  1. 打开程序窗口
  2. 在"选择架构"组中选择你的处理器类型（一般是 x64）
  3. 在"选择通道"组中选择 Chrome 版本（通常选 stable 稳定版）
  4. 确保"运行模式选择"中勾选的是"模式0: 查询链接"
  5. 点击右下方的"开始执行"按钮
  6. 等待程序完成查询（通常只需要几秒钟）

执行完毕后会看到什么：

程序会在日志窗口显示：

  • ✅ 查询到的最新版本号（比如 145.0.7632.110）
  • ✅ Chrome 安装程序的文件大小（比如 431.89 MB）
  • ✅ 文件的 SHA256 校验值（用来验证文件完整性）
  • ✅ 几个可用的下载链接和各个源的位置

同时程序还会生成一个文本文件（比如 stable_x64.txt），
里面保存了最优的下载链接，你可以复制这个链接
用其他工具下载，或者保存下来供以后参考。

📥 模式 1：查询并自动下载 ⭐ 推荐新手用户

这个模式的用途：

这是最直接和最常用的模式。它会自动完成整个过程：

查询版本、测试下载源、自动下载、处理断点续传。如果你只是想快速下载 Chrome 的安装程序，这就是最简单的选择。适合：

  • 👤 想要最简单使用体验的普通用户
  • 🏢 需要在多台电脑上下载相同版本 Chrome 的企业用户
  • ⏱️ 没时间手动处理复杂过程的用户
  • 💾 想要离线保存 Chrome 安装程序的用户

具体怎么使用：

  1. 打开程序
  2. 在"选择架构"中选择你的处理器类型
  • 大多数现代 Windows 电脑都是 x64 架构，这是推荐选项
  • 如果你的电脑很老或是 32 位系统，选 x86
  • Windows on ARM 设备选 arm64
  1. 在"选择通道"中选择版本类型（通常选 stable）
  2. 在"运行模式选择"中勾选"模式1: 查询下载"
  3. 指定下载位置（可以点右边的"选"按钮用文件夹选择器）
  • 如果不改，会下载到程序目录下的 /Download_installer/ 文件夹
  1. 点"开始执行"按钮
  2. 耐心等待下载完成

下载过程中会看到：

  • 🔍 程序先会查询最新版本信息
  • ⚡ 自动测试各个下载源的速度（如果启用了测速功能）
  • 📝 显示将要下载的文件名称和大小
  • 💫 实时显示下载进度（已下载 MB 数 / 总大小）
  • ✅ 下载完成后会显示"下载完成"的提示

程序还会自动处理：

  • 🔄 如果网络中断，下次运行时会自动续传（从上次中断的位置继续）
  • ✔️ 验证本地文件完整性，避免重复下载

🖥️ 处理器类型该选哪个

如果你不确定自己的电脑是什么架构，建议直接选 x64，因为现在几乎所有主流 Windows 电脑都是这个架构。

类型 详细说明 适用情况
x64 64 位处理器架构，这是现代电脑的标准 2015 年之后的所有新电脑，推荐选这个
x86 32 位处理器架构 非常老旧的电脑，或特殊的 32 位 Windows 系统
arm64 ARM 架构 64 位处理器 Windows on ARM 设备（比较少见的特殊系统）

如何确认自己的架构：

  • 按 Win + R 打开运行窗口
  • 输入 msinfo32 后按 Enter
  • 在弹出的系统信息窗口中找到"系统类型"
  • 如果显示"x64"就选 x64，显示"x86"就选 x86
  • 如果不确定，直接选 x64 就没问题

📌 版本通道是什么意思

Chrome 由 Google 维护着多条平行的版本线，每条线的更新频率、稳定性、新功能数量都不同。选择合适的通道能让你获得想要的体验。

通道 中文名 稳定性 更新频率 新功能 适合什么人
stable 稳定版 ⭐⭐⭐⭐⭐ 最稳定 每周一次 已充分测试 👥 所有普通用户，推荐首选
beta 测试版 ⭐⭐⭐⭐ 较稳定 每周一次 部分新功能 🎯 想体验新功能但希望比较稳定的用户
dev 开发版 ⭐⭐⭐ 一般 每天更新 大量新功能 👨‍💻 开发者和技术爱好者
canary 金丝雀版 ⭐⭐ 不稳定 每天多次 最新功能 🧪 技术极客用来测试的

怎么选择：

  • 如果你就是想要一个正常用的 Chrome，选 stable（稳定版）
  • 如果你喜欢尝鲜但又不想太折腾，选 beta（测试版）
  • 如果你是开发者或者特别技术宅，可以试试 dev 或 canary
  • 公司和企业用户一定要选 stable

⚙️ 配置文件详解

CLU_Config.ini 是一个纯文本配置文件，用任何文本编辑器（记事本、VS Code 等）都能打开。文件中的设置会影响程序的默认行为。

🎯 最常用的设置

这些是你最可能需要改的配置项。

网络部分功能的配置：

[Network]
;代理地址,需包含协议头(例如http://127.0.0.1:10808)
proxy=默认为空
;是否测速(1开启,0关闭)
ping_switch=1
;测速超时(毫秒)
ping_timeout=2000

发送查询功能的配置：

[Cxml]
;指定架构:x64,x86,arm64
arch=x64
;版本通道:stable,beta,dev,canary
channel=stable
;构造查询请求的自身版本(通常不用改)
upr_v=1.3.36.151
;构造查询请求的系统版本(可以在uupdump.cn查询)
win_v=10.0.22635.5305
;构造查询请求发去的字符语言代码(实在想改搜:HTML语言代码参考手册)
loc_v=de
;构造查询请求的UserAgent(正常也不用去动)
uUA=Google Update/1.3.36.152;winhttp

工作任务部分的配置：

;是否开启下载模式(0仅查询,1查询并下载,2下载解压释放Chrome++)
runMode=1
;下载文件位置(默认是相对位置写法俩前后的斜杠别丢了)
dir=/Download_installer/
;解压文件位置(默认相对位置)
extdir=/App_Chrome/
;解压临时目录(默认相对位置)
tempdir=/extract_temp/
;Chrome++项目地址(构造的查询Github下载地址从这来)
gcp=https://github.com/Bush2021/chrome_plus

问：程序怎样设置为后台静默运行？

改配置文件就行。

[Settings]
ShowForm=0

这样下次运行程序时就不会弹出窗口了，程序在后台自动执行。
日志输出也不会显示（如果你需要查看，可以稍后改回 ShowForm=1）。

如果想恢复窗口显示，改为：

[Settings]
ShowForm=1
